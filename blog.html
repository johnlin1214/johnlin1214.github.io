JavaScript's Call, Apply, and Bind

By John

JavaScript's Call, Apply, and Bind methods are very useful when you want to change the context an object executing a method. Call, Apply, and Bind can be confusing at first to grasp and sometimes I still trip myself up. Here is my explanation to try to shed some light on these methods. I figured the best way to understand these methods is to see a simple example. I would highly recommend opening up your Dev Tools/Firebug in your Chrome/Firefox browser and follow along with the code. 

The first stepping stone is to understanding the context of "this" in javascript. A function runs within the scope of the object it sits in. In the case below, foo will run in the context of the window object.
function foo() {
  console.log(this) // "this" being the context that the function foo sits in.
}

foo(); // will console log the window object => Window {top: Window, window: Window, location:....
window.foo(): // window.foo() is the same as calling foo() since foo() is a method of the window object.

Let's create an object "john" with the same method foo()
var john = {
  foo: function() {
    console.log(this);
  }
}

john.foo(); // will console log the object "john" - "this" is now in the context of john instead of the window object.

Remember the value of "this" can change depending on the object calling the method. However lets say I wanted to run john.foo() in the context of the window - instead of the john object it sits in. This is where the methods call and apply comes in handy. When I use call or apply, I change the context that "this" executes in.
john.foo.call(window); // Using the Call method, I can execute john.foo() in the context of the window object. Function will now console log the window object.

Call and Apply are the same thing except the main difference is that apply lets you invoke the function with arguments as an array. Call requires the parameters be listed explicitly.
call => .call(context, argument1, argument2, argument3...)
apply => .apply(context, array)

For example:
var john = {
  foo: function(name1, name2, name3) {
    console.log(this);
    console.log("John's friends: " + name1 + ", " + name2 + ", " + name3);
  }
}

john.foo.call(window, "Clifford", "Bella", "Ricky"); // This context = window object and here I explicitly list my parameters.
john.foo.apply(window, ["Clifford", "Bella", "Ricky"]); // This context = window object and here I pass in an array as my parameter.

Both call and apply methods above will console log: "John's friends: Clifford, Bella, Ricky"
Bind takes one argument, the context, and returns the bound function
var johnsBoundFoo = john.foo.bind(window); // so now i created a function johnsBoundFoo that will run foo in the context of the window with the help of bind.
johnsBoundFoo(); // will console log the window object
john.foo(); // will console log the john object, foo is called in the context of the john object

That's Call, Apply, and Bind. If you're new to this, it may be unclear on why you would want to change the context. All you should know for now is that you can change the context. Hope this helps!

JavaScript's Prototype: A quick look

By John

My explanation: The prototype property in JavaScript allows you to add properties and methods to an object. Let’s say you have object constructor Person as shown below. However later on you decided that you wanted all objects created from the object constructor to inherit a certain property or function. I like think of the prototype method as a child inheriting properties from his/her parents. 


function Person(name) {
  this.name = name;
}


var john = new Person("Johnny Appleseed");
console.log(john.name); // "Johnny Appleseed"
Here I created an object john with the Person constructer.
Here’s where prototype comes into play. The prototype extends the method speak and property legs to all instances of the Person constructer. When an object is constructed, it inherits all of the properties of its constructor’s prototype.

Person.prototype.favoriteSauce = "Apple Sauce";
Person.prototype.talk = function() { alert("I love apples!")};

console.log(john.favoriteSauce); // "Apple Sauce"
john.talk(); // "I love apples!"

var sally = new Person("Sally Appleseed");
console.log(sally.favoriteSauce); // "Apple Sauce"
sally.talk(); // "I love apples!"

Linked List Complexity

By John

Linked Lists are one approach to producing a dynamically sized collection of elements. One example lists can have an advantage over arrays is because insertion only requires the previous node to point to the newly inserted node and the newly inserted node to point to the next node. In an array an insertion requires all the subsequent elements to be copied and shifted over one index. Below is a look at the complexity of some linked list methods.

SinglyLinkedList#addToFrontOfList(data) Best-case complexity: O(1). Worst-case complexity: O(1). Inserting a node at the front of a linked list only requires its head to be changed. The rest of the nodes in the list will keep their values and next pointers.

SinglyLinkedList#addToEndOfList(data) Best-case complexity: O(1). Worst-case complexity: O(n). Worst will take O(n) since the entire linked list needs to be transversed to the end to insert the new node.  However, O(1) can be achieved If the tail of a list is known and stored - meaning no traversing the linked list needed and only altering the next pointer of the existing tail.

SinglyLinkedList#find_data_point(data) and SinglyLinkedList#get(index) Best-case complexity: O(n). Worst-case complexity: O(n) . In order to find a particular data point or index of a linked list, the linked list must be traversed - elements of a linked list cannot be “indexed” into.

SinglyLinkedList#delete(data) Best-case complexity: O(n). Worst-case complexity: O(n). Removing the last node of a list takes O(n) time because it require the linked list to be traversed and for the previous node to point to the next node of the deleted node.

SinglyLinkedList#sizeOfLinkedList Best-case complexity: O(1). Worst-case complexity: O(n). In order to find the number of nodes in a linked list, the worst case will take O(n) since the entire linked list needs to be traversed to count each node. To achieve O(1) complexity, the size of the linked list can be stored and updated on each node insertion or deletion - retrieving this size would then be constant time.

ECMAScript 5 Array Methods

By John

ECMAScript 5 defines five iterative methods for arrays that are quite useful. Each of the methods accepts two arguments: a function to run on each item and an optional scope object in which to run the function (affecting the value of this). Below are examples of the 5 iterative methods: every(), some(), filter(), map(), and forEach().

  • every — Runs the given function on every item in the array and returns true if the function returns true for every item.
  • some — Runs the given function on every item in the array and returns true if the function returns true for any one item.
  • filter — Runs the function on every item in the array & returns an array of all items for which the function returns true.
  • map — Runs the given function on every item in the array and returns the result of each function call in an array.
  • forEach — Runs the given function on every item in the array. This method has no return value.

var coolAnimalsArray = ["sloth","honey badger","seal"];
var primeArray = [2,3,5,7,11,13];

coolAnimalsArray.every(function(animal){return animal.length > 3}) // Returns true
primeArray.every(function(num){return num > 3}) // Returns false

coolAnimalsArray.some(function(animal){return animal.length > 20}) // Returns false
primeArray.some(function(num){return num > 12}) // Returns true

coolAnimalsArray.filter(function(animal){return animal.length > 4}) // Returns [ 'sloth', 'honey badger' ]
primeArray.filter(function(num){return num > 4}) // Returns [ 5, 7, 11, 13 ]

coolAnimalsArray.map(function(animal){return "I love " + animal})); // Returns [ 'I love sloth', 'I love honey badger', 'I love seal' ]
primeArray.map(function(num){return num * 2 }) // Returns [ 4, 6, 10, 14, 22, 26 ]

coolAnimalsArray.forEach(function(animal){return primeArray.push(animal)}); // Returns [ 2, 3, 5, 7, 11, 13, 'sloth', 'honey badger', 'seal' ]
